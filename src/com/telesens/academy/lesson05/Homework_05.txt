1) Создать публичный репозиторий на GitHub  
	в который будете складывать выполненные задания
	
2) В своей рабочей папке workflow создать файл git.txt, 
	в который скопировать ссылку на репозиторий
	
3) Инициировать локальный репозиторий:
	- создать файл AD.txt c содержимым:
			a->master
		добавить его в подготовленные файлы
		сделать коммит "A" в ветку мастер
		
	- создать файл BC.txt c содержимым:
			b->master
		добавить его в подготовленные файлы
		сделать коммит "B" в ветку мастер
		
	- добавить в конец файла BC.txt строчку
			c->master
			
		добавить его в подготовленные файлы
		сделать коммит "С" в ветку мастер
			
	- создать ветку bug123
	- переключиться на ветку bug123
		- создать файл 1.txt c содержимым:
				1->bug fix
		- закоммитить "1"
		- добавить в конец файла AD.txt новую строку
				2->bug fix
		- закоммитить "2"
				
	- вернуться к коммиту "C" на ветке мастер
		- добавить в конец файла AD.txt новую строку
				D->master
		- закоммитить "D"
		
	- переключиться на ветку bug123
		- добавить в конец файла ВС.txt новую строку
				3->bug fix
		- закоммитить "3"
		
	- сделать rebase c веткой master
	- перейти на ветку мастер
	- объединить с веткой bug123
	- удалить ветку bug123
	
(Выполнить все действия из командной строки, со среды разработки или используя клиент с графическим интерфейсом)

4) 
	a) Создать новый java проект со следующей структурой, пакетов:
	
        src-com-academy-telesens
                        |-lesson1 
					    |-lesson2
					    |-lesson3
					    |-lesson4
					    |-lesson5
					    ...
	б) Настроить файл: .gitignore				
	c) Закоммитить и выложить в удаленный репозиторий созданный проект
	d) Выполнять задания к каждому уроку в соответствующем пакете lessonX и выкладывать в удаленный репозиторий
